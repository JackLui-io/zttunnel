# 管理员配置说明

## 配置文件位置

管理员邮箱配置从固定路径 `/var/www/java/admin-config.properties` 读取，不再从数据库查询。

## 配置文件格式

在 `/var/www/java/` 目录下创建 `admin-config.properties` 文件，格式如下：

```properties
# 管理员邮箱（支持多个，用逗号分隔）
admin.emails=admin@example.com,admin2@example.com

# 管理员手机号（可选，支持多个，用逗号分隔）
admin.phones=13800138000,13900139000
```

## 配置示例

参考 `admin-config.properties.example` 文件，复制并修改为 `admin-config.properties`。

## 使用说明

1. **部署时**：将 `admin-config.properties` 文件放置在 `/var/www/java/` 目录下
2. **配置修改**：修改配置文件后，需要重启应用才能生效
3. **日志查看**：应用启动时会输出配置文件加载日志，包括：
   - 配置文件路径（固定为 `/var/www/java/admin-config.properties`）
   - 加载的管理员邮箱数量
   - 加载的管理员手机号数量

## 注意事项

- 配置文件路径固定为 `/var/www/java/admin-config.properties`（Ubuntu 22.04系统）
- 如果配置文件不存在，系统会尝试从系统属性 `admin.email` 读取默认邮箱
- 如果未配置管理员邮箱，设备掉线通知将无法发送
- 配置文件支持热更新（调用 `adminConfig.reload()` 方法）
- 确保应用有读取该文件的权限（建议使用 `chmod 644 /var/www/java/admin-config.properties`）
